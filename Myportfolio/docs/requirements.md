# PC構成提案Webアプリ 要件定義書

## 1. 目的
ユーザーが予算と用途（ゲーミングPC、動画編集PC、汎用PC）を入力すると、互換性を考慮した最適なPC構成を自動提案し、購入リンクを表示する。

## 2. スコープ
本システムは以下を対象とする。

- 予算・用途に応じたPC構成自動提案
- 主要PCパーツの価格情報収集とキャッシュ
- 提案結果の可視化と購入導線提供

以下は将来拡張として扱う。

- ユーザー登録およびお気に入り保存
- 性能指標（PassMark、ベンチマーク）表示
- 在庫状況チェック

## 3. 機能要件

### 3.1 必須機能

#### 3.1.1 予算入力機能
- ユーザーは日本円で予算を入力できる。
- 入力値は整数値で扱い、最小値・最大値をバリデーションする。

#### 3.1.2 用途選択機能
- ユーザーは以下から用途を選択できる。
- `gaming`（ゲーミングPC）
- `video_editing`（動画編集PC）
- `general`（汎用PC）

#### 3.1.3 構成生成機能
- 予算配分アルゴリズムに基づき、パーツ候補を選定する。
- 互換性チェックを実施する。
- CPUソケット整合性
- メモリ規格整合性
- 電源容量余裕（推奨W数を満たす）
- ケースサイズ整合性

#### 3.1.4 パーツ情報取得機能
- 価格ドットコムを主要データソースとして価格・商品情報を収集する。
- 必要に応じてAmazon/Rakutenの購入リンクを表示する。
- 取得データはDBにキャッシュし、API応答で再利用する。

#### 3.1.5 結果表示機能
- 提案構成として以下を表示する。
- CPU
- GPU
- メモリ
- ストレージ
- 電源
- ケース
- 価格合計を表示する。
- 各パーツの購入リンクを表示する。

### 3.2 追加機能（将来的に）
- ユーザー登録・ログイン
- お気に入り構成の保存
- 性能指標（PassMark、各種ベンチマーク）表示
- 在庫状況チェック

## 4. 非機能要件

### 4.1 パフォーマンス
- 構成生成APIの応答時間は3秒以内を目標とする（通常負荷時）。

### 4.2 スケーラビリティ
- 同時アクセス100ユーザー程度に耐える構成とする。

### 4.3 セキュリティ
- 通信はHTTPSを必須とする。
- API認証はJWTを採用する。

### 4.4 可用性
- 稼働率99%以上を目標とする。

## 5. 技術要件
- フロントエンド: React + TailwindCSS
- バックエンド: FastAPI（Python）
- DB: PostgreSQL（パーツ情報キャッシュ）
- スクレイピング: Python（BeautifulSoup、Requests）、必要時Playwright
- 構成生成アルゴリズム: Python（PuLP または OR-Tools）
- ホスティング候補:
- AWS（EC2 + RDS）
- Vercel + Supabase

## 6. インターフェース要件

### 6.1 UIフロー
1. ホーム画面
2. 予算入力フォーム
3. 用途選択
4. `構成を生成` ボタン押下
5. 結果画面（パーツ一覧 + 購入リンク）

### 6.2 API設計（初期案）

#### POST `/generate-config`
入力例:

```json
{
  "budget": 100000,
  "usage": "gaming"
}
```

出力例:

```json
{
  "usage": "gaming",
  "budget": 100000,
  "total_price": 98700,
  "parts": [
    {
      "category": "cpu",
      "name": "Example CPU",
      "price": 24800,
      "url": "https://example.com/cpu"
    },
    {
      "category": "gpu",
      "name": "Example GPU",
      "price": 42800,
      "url": "https://example.com/gpu"
    }
  ]
}
```

## 7. 制約条件
- 価格ドットコムの利用規約を遵守する。
- スクレイピング頻度を制御する（レート制限、アクセス間隔、リトライ制御）。
- 金額表記・入力は日本円対応とする。
- PCパーツ情報は1日1回を目安に定期更新する。

## 8. 開発ステップ
1. 構成生成アルゴリズム試作（仮データ）
2. スクレイピング実装（CPU/GPUから開始しDB保存）
3. FastAPIでAPI構築
4. ReactでUI開発
5. 統合テストとデプロイ

## 9. 受け入れ条件（初版）
- 指定予算と用途で、互換性エラーのない構成が返ること。
- 返却構成の価格合計が予算を大幅超過しないこと。
- 各パーツに購入リンクが含まれること。
- 通常時に3秒以内でレスポンスを返すこと。
